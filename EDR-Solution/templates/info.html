1. Bulk File Delection
SEVERITY => LOW
This alert arise when the EDR identifies large amount of file being deleted suddenly. To be specific more than 10 files are deleted suddenly.
In the alert you can also see the process which as cause this alert to raise.
2. Log File Deletion
SEVERITY => CRITICAL
This alert arises when an attacker attempts to delete log files to clear their tracks. Since log files contain crucial records of system activities, their deletion could indicate a malicious attempt to evade detection.
In the alert, you can also see the process responsible for triggering this alert.
3. High CPU Usage Spike
SEVERITY => MEDIUM
This alert arises when the CPU usage goes beyond the threshold value, which can be adjusted by the user on the settings page.
The alert also displays the top 5 processes consuming the maximum CPU resources, helping to quickly identify the cause of the spike.
4. Insecure Connection Detected
SEVERITY => CRITICAL
This alert arises because the EDR continuously checks all IPs connected to the system, and if it identifies a malicious IP address, it generates this alert.
Response Steps:
  • Before going ahead verify the IP's reputation by yourself to avoid False-Positive.
  • Disconnect from the internet immediately to isolate the threat.
  • Block the malicious IP address using Windows Firewall:
    1. Open Windows Defender Firewall with Advanced Security.
    2. Go to "Inbound Rules" or "Outbound Rules".
    3. Click "New Rule" and select "Custom".
    4. Under "Which remote IP addresses does this rule apply to?" select "These IP addresses" and add the malicious IP.
    5. Choose "Block the connection" and complete the wizard.
